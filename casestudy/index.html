MY ROLE

I worked individually to develop a web-based application with HTML, CSS, and JavaScript.

CONTEXT & CHALLENGE

BACKGROUND

I learned to build a unique application that takes advantage of gestures and accelerometers. Special consideration was given to limited screen real estate, low bandwidth internet access, no internet access, and touch screen devices.

GOALS

  • Learn new techniques along with the best practices in mobile app design and development
  • Have a comprehensive understanding of designing and developing interactive web applications for modern mobile devices
  • Successfully build out a fully functional mobile app by focusing on design and user experience

PROCESS & INSIGHT

CHOOSING THE IDEA

My goal was to choose an application with one core function. I decided on an interval timer because it is a tool that I consistently use. I started with brainstorming a potential layout by sketching out the screens. Next, I pulled inspiration from existing apps and styles that I could include in my interface. I had to keep in account the screen real estate as well as hit sizes when using mobile.

DEVELOPMENT

ALPHA

My main priority for the alpha was making sure I was able to code out all of the screens for the web app and to also make sure the buttons are functional. I implemented a modal that opens once you hit the start button.

I was also able to incorporate the "add to home screen" functionality using JavaScript that alerts the user a couple seconds after they visit the page. I decided that due to feasibility, I was going to focus on getting the timer to function rather than the progress bar. The second image just consists of a placeholder, so my next focus was coding it out.

Lastly, I added a screen that alerts the user to turn the screen back to portrait mode if they were to flip their phone to landscape.

BETA

The focus for the beta was adding functionality to the timer and making sure the input matched the output. There was some troubleshooting involved when it came to making sure the timer was functioning properly. This mainly consisted of fixing the JavaScript so that when the user hits start, the correct settings would appear in the timer. I also solved the issue of having the timer skip numbers or going too fast by removing some lines of code out of the loop and incorporating a reset function that reset the settings once the user quit out of the modal.

RESULTS

FINAL

The biggest focus for the final was implementing localStorage functionality, so that the user could quit out of the app and open it again with the same settings they chose before. Because the settings consisted of numbers, there was a challenge of converting the strings into interval variables after checking local storage for information. This consisted of loops that read the data and then wrote it, and I worked to debug this by checking the local storage in the browser until the changes I made starting working.

After fixing the issue of the timer and implementing localStorage, the timer was functioning properly and the final project was completed. The app could successfully be downloaded to the home screen, function properly, and save user input once closed and reopened.
